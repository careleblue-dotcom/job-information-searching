## 一、项目背景与目标

### 1.1 项目背景
马克思主义学院学生的就业方向高度集中在体制内岗位，包括：
- 党政机关公务员
- 选调生项目
- 人才引进计划
- 党政基地项目
- 事业单位编制岗位

目前学生获取这些信息需要手动访问多个网站，效率低下且容易遗漏。

### 1.2 项目目标
开发一个微型就业信息筛选工具，实现：
1. **自动采集**：定期从目标网站抓取招聘信息
2. **智能筛选**：按岗位类型、地区、截止日期等维度筛选
3. **便捷展示**：网页端展示，支持搜索和分类浏览
4. **及时推送**：重要信息及时通知（后期扩展）

### 1.3 目标受众
- **当前阶段**：马克思主义学院学生
- **未来扩展**：全校对体制内岗位感兴趣的学生

## 三、技术架构设计

### 3.1 整体架构

```
┌─────────────────────────────────────────────────┐
│                   用户浏览器                      │
└─────────────────┬───────────────────────────────┘
                  │ HTTPS
                  ▼
┌─────────────────────────────────────────────────┐
│          前端展示层（Web页面）                    │
│    - 招聘信息列表                                 │
│    - 筛选器（地区、类型、日期）                   │
│    - 搜索功能                                     │
└─────────────────┬───────────────────────────────┘
                  │ 读取
                  ▼
┌─────────────────────────────────────────────────┐
│            数据存储层                             │
│    - JSON文件 或 SQLite数据库                     │
│    - 存储结构化招聘信息                           │
└─────────────────┬───────────────────────────────┘
                  │ 写入
                  ▼
┌─────────────────────────────────────────────────┐
│          后端爬虫层（Python脚本）                  │
│    - 定时任务（每日爬取）                         │
│    - 数据清洗与结构化                             │
│    - 去重与更新                                   │
└─────────────────────────────────────────────────┘
```

### 3.2 技术栈选择

#### 后端爬虫
- **语言**：Python 3.9+
- **核心库**：
  - `requests` - HTTP请求
  - `BeautifulSoup4` - HTML解析
  - `lxml` - 高性能XML/HTML解析
  - `pandas` - 数据处理
  - `Playwright` - 浏览器自动化（用于难爬的网站）

#### 数据存储
- **MVP阶段**：JSON文件（简单易部署）
- **扩展阶段**：SQLite（轻量级，无需服务器）

#### 前端展示
- **框架**：Vue 3 或 React（轻量级）
- **UI组件**：Element Plus 或 Ant Design Vue
- **部署**：Cloudflare Pages 或 Vercel（免费）

#### 定时任务
- **方案A**：GitHub Actions（免费，每日定时运行爬虫）
- **方案B**：本地cron任务（如果有常开电脑）

## 四、产品功能规划

### 4.1 MVP功能（第一阶段）

**核心功能**：
1. ✅ 招聘信息列表展示（按发布时间倒序）
2. ✅ 按类别筛选（选调生/人才引进/事业单位/公务员）

4. ✅ 关键词搜索

**非功能性需求**：
- 数据每日自动更新
- 页面加载速度 < 2秒

1. ⭐ 截止日期提醒（微信/邮件通知）
2. ⭐ 个性化推荐（根据用户画像）
3. ⭐ 数据统计看板（各地区招录人数、热门专业等）
4. ⭐ 导出功能（导出为Excel/PDF）
5. ⭐ 用户反馈与纠错

## 六、部署方案

### 6.1 推荐方案：GitHub + Cloudflare Pages（完全免费）

**优势**：
- ✅ 完全免费
- ✅ 无需自己维护服务器
- ✅ GitHub Actions自动运行爬虫
- ✅ Cloudflare CDN全球加速
- ✅ 自动HTTPS

**实施步骤**：
1. 代码托管到GitHub仓库
2. 爬虫脚本作为GitHub Actions定时任务（每日运行）
3. 爬取的数据提交回仓库（或存储到Cloudflare KV）
4. 前端页面部署到Cloudflare Pages
5. 绑定自定义域名（可选）

### 6.2 备选方案：轻量云服务器

如果有预算（约30-50元/月），可以租用轻量应用服务器：
- 腾讯云轻量应用服务器（2核2G，约30元/月）
- 阿里云轻量应用服务器（同等配置）

优势：完全自主控制，可以实现更复杂的后台功能。

## 七、风险与应对

### 7.1 技术风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 目标网站改版 | 爬虫失效 | 模块化设计，单独维护各网站爬虫 |
| 反爬机制增强 | 无法获取数据 | 使用Playwright，添加代理池 |
| 数据量过大 | 性能下降 | 分页加载，添加缓存机制 |

### 7.2 法律风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 违反网站服务条款 | 被封禁IP | 遵守robots.txt，控制请求频率 |
| 数据使用合规 | 法律纠纷 | 只展示标题和链接，不转载全文 |

**重要提醒**：
- 爬虫开发需遵守各网站的`robots.txt`协议
- 控制请求频率，避免对目标网站造成压力
- 只采集公开信息，不采集需要登录的私密信息
- 本工具仅供学习交流使用，不得用于商业目的

## 十、总结与建议

### 10.1 核心建议

1. **先做容易的**：不要一开始就挑战最难爬的网站，先把流程跑通
2. **MVP思维**：第一版产品不需要完美，能用就行，逐步迭代
3. **善用免费资源**：GitHub + Cloudflare完全够用，不需要花钱买服务器
4. **遵守规则**：爬虫要讲武德，别把人家网站搞崩了

### 10.2 时间预估

- **最小可用版本（MVP）**：2-3周
- **功能完善版本**：6-8周
- **全网站覆盖版本**：10-12周

### 10.3 成功标准

- ✅ 能够自动采集至少3个网站的数据
- ✅ 网页能够正常展示招聘信息
- ✅ 至少有10个同学在使用并给出正面反馈
- ✅ 数据更新及时（延迟不超过24小时）
