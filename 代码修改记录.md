## 1. 项目配置文件

### `config.py` — 中心配置模块

**作用**：所有爬虫共享的配置参数

**内容**：
- `PROJECT_ROOT` / `DATA_DIR`：项目根目录和数据目录自动检测
- `HEADERS`：HTTP 请求头（模拟 Chrome 浏览器）
- `DELAY = 1`：请求间隔 1 秒
- `CENTRAL_GOV`：中央国家机关平台基础 URL + 三个栏目（zpgg/gxbyszpzl/zytz）及其中文名
- `JSZZB`：江苏省委组织部官网 URL + 关键词列表（选调/公务员/人才引进/编制/党政/遴选/优培）
- `REGIONS`：34 个省级行政区域列表
- `OUTPUT_FILES`：输出文件路径（all_jobs.json / central_gov_jobs_full.json）

### `requirements.txt` — Python 依赖


## 2. 爬虫工具模块

### `shared.py` — 通用工具函数（248 行）

**核心函数**：

| 函数 | 说明 |
|------|------|
| `fetch_page(url)` | 通用 HTTP 请求，返回 HTML 文本 |
| `parse_date(date_str)` | 日期解析，支持 `YYYY-MM-DD` 和 `YYYY年MM月DD日` 格式 |
| `extract_region(title, content, source)` | 从文本中匹配 31 个省级行政区，返回地区名 |
| `extract_category(title, content)` | 智能分类检测（选调生/人才引进/事业单位/公务员考试/高校毕业生/补充公告/招聘信息/其他） |
| `crawl_job_detail(url, source_name)` | 通用详情页爬取：多选择器提取标题/日期/正文/单位/地区/分类 |
| `save_and_merge(new_data, output_file)` | 保存并去重合并（按 URL 去重） |
| `save_json(data, filepath)` | 简单 JSON 覆盖写入 |
| `batch_crawl(urls_with_source, save_interval, output_file)` | 批量爬取，每 10 条增量保存 |

**设计特点**：
- 多选择器 fallback 机制（标题/日期/正文各有 5+ 种选择器）
- 异常捕获不中断流程
- 增量保存防中断丢数据

## 3. 中央国家机关平台爬虫

### `crawler_central_gov.py` — 中央国家机关事业单位招聘平台（288 行）

**目标网站**：http://114.255.111.180/SYrlzyhshbzb/fwyd/SYkaoshizhaopin/zyhgjjgsydwgkzp/

**功能**：
- 自动爬取 3 个栏目：zpgg（招聘信息）、gxbyszpzl（高校毕业生招聘专栏）、zytz（补充公告）

- URL 构建逻辑：处理绝对路径 `http://`、相对路径 `./`、根路径 `/` 三种情况

**核心函数**：

| 函数 | 说明 |
|------|------|
| `crawl_category(category_code, category_name, limit=20)` | 爬取单个栏目，支持 limit 限制条数 |
| `crawl_job_detail(url, category)` | 详情页解析（本文件重写版本，增加 organization/deadline 提取） |
| `main()` | 主入口，遍历三个栏目并汇总保存 |

**数据字段**：
```json
{
  "title", "url", "publish_date", "content",
  "category", "category_code",
  "organization", "deadline",
  "crawl_time"
}
```

## 7. 数据清理工具

### `cleanup.py` — 完整数据清理工具（242 行）

**三种运行模式**：

| 模式 | 命令 | 功能 |
|------|------|------|
| audit | `python cleanup.py audit` | 只读审查：异常标题/内容过短/分类分布/地区分布/重复标题/乱码检测 |
| round1 | `python cleanup.py round1` | 第一轮清理：URL 去重 + region 修复（优先匹配 source） + category 智能修复 |
| round2 | `python cleanup.py round2` | 第二轮清理：标题去重 + 内容过短过滤 |
| all | `python cleanup.py all` | 全流程：round1 → round2 串行执行 |

**清理过程数据变化**：151 条 → 96 条有效数据
- 删除 55 条问题数据：空标题(19)、内容过短/乱码(22)、重复(14)
- region 修复：优先从 source 字段匹配省份
- category 修复：智能关键词匹配

**自动备份机制**：每次清理前自动备份到 `data/all_jobs_backup_YYYYMMDD_HHMMSS.json`

## 8. 前端页面

### `index.html` — Vue 3 单页应用（404 行）

**技术栈**：Vue 3.4（CDN 加载，无构建步骤）

**UI 设计**：
- 主色调：南大紫 `#72237E`
- 院徽：`images/logo.png`（南京大学马克思主义学院）
- 标题：南京大学马克思主义学院就业信息网
- 子标题：聚焦党政机关 / 选调生 / 人才引进 / 编制岗位

**功能模块**：

| 功能 | 实现方式 |
|------|---------|
| 关键词搜索 | v-model 绑定，实时过滤（标题/单位/内容） |
| 分类筛选 | 按钮组，动态从数据生成分类列表 |
| 地区筛选 | 下拉框（31 个省级行政区 + 全部），点击外部关闭 |
| 统计信息 | 实时显示匹配条数 + 当前分类/地区 + 页码 |
| 招聘卡片 | 显示标题/分类标签/日期/单位/地区/内容摘要 |
| 详情弹窗 | 模态框展示完整信息 + 原始链接 |
| 分页 | 每页 20 条，显示页码范围 |
| 空状态 | 无匹配数据时显示提示 |
| 防缓存 | fetch URL 加时间戳参数 + Cache-Control meta |
| 空值保护 | 所有模板表达式加 `|| ''` 保护，防止渲染崩溃 |

**关键修复记录**：
1. 数据源从 `central_gov_jobs_full.json`（30 条）改为 `all_jobs.json`（96 条）
2. 地区筛选从按钮改为下拉框（31 个固定省份列表）
3. 前端补充缺失 region 字段（遍历省份关键词匹配）
4. 修复 `job.content.substring(0,150)` 在 content 为 null/undefined 时的 JS 错误（加 `|| ''` 保护）
5. 修复 HTML 模板多 `</div>` 导致 Vue 挂载失败 bug
